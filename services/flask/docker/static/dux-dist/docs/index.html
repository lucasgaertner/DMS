dux

The most valuable ready-to-use framework around the d.velop group.

DUX (d.velop user experience) offers on the basis of HTML, CSS and JavaScript different templates for typography or buttons. The basic thought is to bring developers and designers closer together. DUX can be compared with a tool-box from which the structure and the styling of the Web-Frontends can be built or even optimised. Above all DUX stands for for responsive web design where the composition of the website occurs dynamically.

Enjoy and have fun!

Installation

To install DUX, run the following command (after configuring bower):

bower install dvelop-dux

Configure bower

To configure Bower to use the dvelop-bower ProGet feed, add this registry to your .bowerrc file:

"registry": "http://nuget.d-velop.de/bower/dvelop-bower"

If your feed does not allow anonymous access, you will need to supply credentials in the URL:

"registry": "http://user:password@nuget.d-velop.de/bower/dvelop-bower"

Include CSS

Please reference the relevant files in the <head> section of your html file.

<!doctype html>
<html>
    <head>
        <title>example</title>
        <link rel="stylesheet" href="css/dux.css" type="text/css" media="screen,projection,print"/>
    </head>
    <body>
        <first-element></first-element>
    </body>
</html>

Include Javascript

To refer the javascript file please use the src attribute in the <script> tag as the last element of the <body>.

<!doctype html>
<html>
    <head>
        <title>example</title>
    </head>
    <body>
        <last-element></last-element>
        <script type="text/javascript" src="js/libs.js"></script>
    </body>
</html>

In combination with IIS

The IIS uses the "static content" module for static files . To let the IIS know what mime type to be sent for which file extension, it has a integrated "MimeMap" With th default settings the entries for .svg- and .woff files that are delivered with DUX are missing.

Please add them to your web.config as follows.

<configuration>
  <system.webServer>
    <staticContent>
      <remove fileExtension=".svg" />
      <remove fileExtension=".woff" />
      <remove fileExtension=".woff2" />
      <mimeMap fileExtension=".svg" mimeType="image/svg+xml" />
      <mimeMap fileExtension=".woff" mimeType="application/font-woff" />
      <mimeMap fileExtension=".woff2" mimeType="application/font-woff2" />
    </staticContent>
  </system.webServer>
</configuration>

Alternatively, add these entries manually at a higher config level. Adding these entries via the web.config is more simple.
