4.1. Definition von Variablen:

4.1.1.Lokale Variablen
Variablen, welche innerhalb eines {}-Blocks einer Datei mittels ,vars” definiert werden, stehen
innerhalb dieses Kontextes lokal zur Verfiigung.

vars 1ReturnValue =0 // Definition eines numerischen Wertes
// max. 255 Zeichen!

vars 1KundenName(35) = "" // Variable welche max. 35 Zeichen aufnehmen
// kann.

vars 1BetragsArr[100] // Array mit max. 100 Elementen

"Hallo" // max. 255 Zeichen
"Welt" // max. 255 Zeichen
uw // max. 255 Zeichen

vars 1TestStringl
vars 1TestString2
vars 1ResultString

vars 1StringLength 0 // max. 255 Zeichen
String-Addition:

1ResultString = 1lTestStringl ## " " ## 1TestString2
ODER

1ResultString = ":lTestStringl :1TestString2"

>,Hallo Welt’

4.1.2.Speichern der Variablen im Workflow-Kontext

Soll eine definierte JPL-Variable innerhalb des Workflows als Steuerungs-Parameter genutzt, sprich
auch in Folge-Schritten genutzt werden, muss diese Variable mit ihrem aktuellen Wert gesichert
werden.

In diesem Zusammenhang spricht man hier auch von ,,Globalisierung”. Dies erfolgt mit der Funktion
call save vars (,,<varname>™). Mit Hilfe dieser Funktion wird die Variable mit ihrem aktuellen
Wert in die DB-Tabelle ,wfl_step_user_values” Ubernommen und steht dann fir weitere Schritte zur
Verfiigung.
