3.5.3. Bearbeitung aktiver Workflows
Muss eine aktive Workflow-Definition angepasst werden, meist im Rahmen kleiner Anderungen bzw.
der Ausbesserung kleinerer Fehler, kann dies in der aktuellen Version vorgenommen werden. Dazu
kénnen dann folgende Schritte ausgefuhrt werden.

 

 

 

 

Abbildung 3.8 Schaubild der Bearbeitung einer aktiven Workflow-Definition

1. Workflow-Definition in die Bearbeitung nehmen. Damit wird die Workflow-Definition in den
Status Bearbeitung Uberfuhrt und folgt damit dem Standard d.3-Status-Mechanismus. Die aktive
/ freigegebene Version ist dabei nattirlich weiterhin aktiv.

2. Nun kann die Version in Bearbeitung ganz normal angepasst werden. Dabei sollte darauf geachtet
werden, da es sich hier um Arbeiten an einem aktiven Workflow handelt, dass sich die
Anderungen eher im Rahmen einer Fehlerbehebung als einer Integration neuer Funktionalitaten
bewegen. Eine Uberfiihrung in eine Test-Version ist nicht méglich, wenn die alte Version im
Status Freigabe / Aktiv ist, da sich die Ausfuhrungsversionen / Workflow-IDs nicht unterscheiden!

3. Wird nun die neue Version aktiviert, werden automatisch alle Dokumente, welche aktuell mit
dieser Workflow-Definition bearbeitet werden, mit der neuen Version weitergefuhrt.

4. Die alte Version wird deaktiviert und ist damit im Status ,,Archiv / blau”.
